1. FRONT-END (Single HTML Page) – 10 marks
✔ Required Features (must appear on the page, using modals or sections)

Registration form (fields + submit)

Login form

User feed section (shows posts only from followed users)

Search section

Search users

Search posts/contents

Post creation area (write/post text)

Follow/unfollow buttons

Profile picture upload (if doing file upload)

✔ Design Rules

Must be one HTML file only

Use HTML + CSS + JavaScript (Vanilla)

Allowed frameworks: Bootstrap, Foundation, or jQuery

NOT allowed: React, Vue, Angular

No alerts → use inline messages or modals

✔ To get full 7.5 + 2.5 marks

Modern attractive design with consistent Bootstrap styling

Polished usability: no broken links/buttons, all sections reachable

Clear layout (Feed in centre, search on top, post modal, etc.)

Professional CSS styling

2. WEB SERVICE (Node.js + Express) – around 25 marks

All your API endpoints must start with your student ID, e.g.:

http://localhost:8080/M01028026/users

✔ USER REGISTRATION

POST /{ID}/users
Receive JSON
Insert into MongoDB
Return JSON result

✔ USER SEARCH
GET /{ID}/users?q=xxx
Query param required
Return JSON list from MongoDB

✔ LOGIN SYSTEM
GET /{ID}/login → return login status
POST /{ID}/login → check credentials in MongoDB
DELETE /{ID}/login → logout user

✔ SESSION MANAGEMENT

Must use Express-session OR JWT OR custom cookie
Must protect:
GET login
DELETE login
POST contents
POST follow
DELETE follow
GET feed
Must not send userID through request body for these.

✔ SOCIAL NETWORKING
POST /{ID}/contents
Store post text in MongoDB

GET /{ID}/contents?q=xxx

Search content by keywords

POST /{ID}/follow or /{ID}/follow/:userid

Store in MongoDB

DELETE /{ID}/follow or /{ID}/follow/:userid

Remove follow relationship

GET /{ID}/feed

Must ONLY show content from followed users

NOT from all users
